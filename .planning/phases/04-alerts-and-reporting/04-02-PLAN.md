  <action>
    1. Create `src/lib/export/csv-serializer.ts`:
       - Export `serializeCsv(params: { headers: string[]; rows: Record<string, unknown>[] }): string`
       - First line: headers joined by comma
       - Each row: map header keys to values, escape each value with `escapeField()` helper
       - `escapeField(value: unknown): string` -- convert to string (null/undefined -> ''), replace `"` with `""`, wrap in double quotes
       - Return header line + '\n' + row lines joined by '\n'

    2. Create `src/app/api/export/snapshots/route.ts`:
       - Export GET handler accepting NextRequest
       - Required query params: `startDate` (YYYY-MM-DD), `endDate` (YYYY-MM-DD) -- return 400 with `{ error: 'startDate and endDate query params are required' }` if missing
       - Query: `db.select({ date: dailySnapshots.date, keyword: keywords.label, brand: brands.name, provider: dailySnapshots.providerId, mentionRate: dailySnapshots.mentionRate, avgPosition: dailySnapshots.avgPosition, avgSentiment: dailySnapshots.avgSentiment, runCount: dailySnapshots.runCount }).from(dailySnapshots).leftJoin(keywords, eq(dailySnapshots.keywordId, keywords.id)).leftJoin(brands, eq(dailySnapshots.brandId, brands.id)).where(and(gte(dailySnapshots.date, startDate), lte(dailySnapshots.date, endDate))).orderBy(dailySnapshots.date)`
       - CSV headers: `['Date', 'Keyword', 'Brand', 'Provider', 'Mention Rate', 'Avg Position', 'Avg Sentiment', 'Run Count']`
       - Map rows to objects with matching keys
       - Call `serializeCsv({ headers, rows })`
       - Return `new Response(csv, { headers: { 'Content-Type': 'text/csv', 'Content-Disposition': 'attachment; filename="snapshots-YYYY-MM-DD.csv"' } })` where YYYY-MM-DD is today's date

    3. Create `src/app/api/export/alerts/route.ts`:
       - Export GET handler accepting NextRequest
       - Optional query params: `startDate`, `endDate` (default to last 30 days if not provided using date-fns)
       - Query: select from alerts with leftJoin on keywords and brands, filter by createdAt date range, order by createdAt desc
       - CSV headers: `['Date', 'Keyword', 'Brand', 'Alert Type', 'Previous Value', 'Current Value', 'Read', 'Notified At']`
       - Map alert rows -- previousValue and currentValue are jsonb, stringify them for CSV
       - Return CSV Response with Content-Disposition header
