    2. Create `src/worker/handlers/weekly-report-handler.ts`:
       - Export `handleWeeklyReportJob(job: Job): Promise<void>`
       - Calculate date range: endDate = yesterday (format YYYY-MM-DD), startDate = 7 days before endDate (use date-fns `subDays` + `format`)
       - Query snapshots in range: `db.select({ date: dailySnapshots.date, keyword: keywords.label, brand: brands.name, provider: dailySnapshots.providerId, mentionRate: dailySnapshots.mentionRate, avgPosition: dailySnapshots.avgPosition }).from(dailySnapshots).leftJoin(keywords, eq(dailySnapshots.keywordId, keywords.id)).leftJoin(brands, eq(dailySnapshots.brandId, brands.id)).where(and(gte(dailySnapshots.date, startDate), lte(dailySnapshots.date, endDate))).orderBy(dailySnapshots.date)`
       - Count alerts in range: `db.select({ count: sql<number>\`count(*)\` }).from(alerts).where(and(gte(alerts.createdAt, startOfRange), lte(alerts.createdAt, endOfRange)))`
       - If snapshots.length === 0, log `'Weekly report: no data in range, skipping'` and return (pitfall 5)
       - Check `isEmailConfigured()` -- if false, log and return
       - Check `process.env.ALERT_EMAIL_TO` -- if empty, log and return
       - Call `renderWeeklyReport(...)` to get subject + html
       - Call `getTransporter().sendMail({ from: process.env.SMTP_FROM || 'SEO AI Monitor <noreply@localhost>', to: process.env.ALERT_EMAIL_TO, subject, html })`
       - Log success: `console.log(\`Weekly report sent for ${startDate} to ${endDate}\`)`
