1. Create `drizzle.config.ts` at project root:

2. Create `src/lib/db/schema.ts` with ALL 6 tables. This is the most critical file in Phase 1.

   **Table 1: keywords**
   - id: text PK with crypto.randomUUID() default
   - label: text NOT NULL (display name, e.g. "Best SEO tools")
   - prompt: text NOT NULL (full prompt text, e.g. "What are the best SEO tools in 2025?")
   - promptVersion: integer NOT NULL default 1 (increment on prompt edit, per DATA-08)
   - isActive: boolean NOT NULL default true (toggle without deleting history)
   - createdAt: timestamp defaultNow NOT NULL
   - updatedAt: timestamp defaultNow NOT NULL

   **Table 2: brands**
   - id: text PK with crypto.randomUUID() default
   - name: text NOT NULL (brand/company name)
   - aliases: jsonb typed as string[] NOT NULL default [] (per CONTEXT.md: JSON array in brands table)
   - isOwn: boolean NOT NULL default false (per CONTEXT.md: is_own flag distinguishes own vs competitor)
   - createdAt: timestamp defaultNow NOT NULL
   - updatedAt: timestamp defaultNow NOT NULL

   **Table 3: queryRuns**
   - id: text PK with crypto.randomUUID() default
   - keywordId: text NOT NULL references keywords.id
   - providerId: text NOT NULL ('chatgpt' | 'perplexity' | 'gemini')
   - model: text NOT NULL (model identifier)
   - prompt: text NOT NULL (exact prompt snapshot at query time)
   - promptVersion: integer NOT NULL (snapshot from keyword at query time)
   - rawResponse: text NOT NULL (per DATA-07: store raw response)
   - citations: jsonb typed as string[] default []
   - tokensUsed: integer nullable
   - costUsd: decimal(10,6) nullable (per DATA-06: cost tracking)
   - latencyMs: integer nullable
   - runNumber: integer NOT NULL (1, 2, or 3 within batch per DATA-05)
   - batchId: text NOT NULL (groups runs from same check cycle)
   - runType: text NOT NULL default 'scheduled' ('scheduled' | 'on-demand')
   - createdAt: timestamp defaultNow NOT NULL
   - Indexes: (keywordId, providerId), (batchId), (createdAt)

   **Table 4: brandMentions**
   - id: text PK with crypto.randomUUID() default
   - queryRunId: text NOT NULL references queryRuns.id
   - brandId: text NOT NULL references brands.id
   - mentioned: boolean NOT NULL default false
   - position: integer nullable (ordinal rank: 1st, 2nd, 3rd)
   - contextSnippet: text nullable
   - sentiment: text nullable ('positive' | 'neutral' | 'negative')
   - isRecommended: boolean default false
   - createdAt: timestamp defaultNow NOT NULL
   - Indexes: (queryRunId), (brandId, createdAt)

   **Table 5: alerts**
   - id: text PK with crypto.randomUUID() default
   - keywordId: text nullable references keywords.id
   - brandId: text nullable references brands.id
   - alertType: text NOT NULL ('appeared' | 'disappeared' | 'rank_changed' | 'sentiment_changed')
   - previousValue: jsonb nullable
   - currentValue: jsonb nullable
   - isRead: boolean NOT NULL default false
   - createdAt: timestamp defaultNow NOT NULL
   - notifiedAt: timestamp nullable

   **Table 6: dailySnapshots**
   - id: text PK with crypto.randomUUID() default
   - date: date NOT NULL
   - keywordId: text NOT NULL references keywords.id
   - brandId: text NOT NULL references brands.id
   - providerId: text NOT NULL
   - avgPosition: decimal(5,2) nullable
   - mentionRate: decimal(5,4) nullable (0.0000-1.0000)
   - avgSentiment: decimal(5,2) nullable
   - runCount: integer NOT NULL default 0
   - createdAt: timestamp defaultNow NOT NULL
   - Index: (keywordId, brandId, providerId, date)

   Use `crypto.randomUUID()` for ID generation (built-in, no extra dependency).
   Import from `drizzle-orm/pg-core`: pgTable, text, boolean, timestamp, integer, decimal, uuid, jsonb, date, index.
   Import `relations` from `drizzle-orm` and define relations for type-safe joins between all tables.

3. Create `src/lib/db/client.ts` with hot-reload-safe singleton:
   ```typescript
   import { drizzle } from 'drizzle-orm/postgres-js';
   import postgres from 'postgres';
   import * as schema from './schema';

   const connectionString = process.env.DATABASE_URL!;

   const globalForDb = globalThis as unknown as {
     db: ReturnType<typeof drizzle> | undefined;
     pgClient: ReturnType<typeof postgres> | undefined;
   };

   const client = globalForDb.pgClient ?? postgres(connectionString);
   export const db = globalForDb.db ?? drizzle(client, { schema });

   if (process.env.NODE_ENV !== 'production') {
     globalForDb.db = db;
     globalForDb.pgClient = client;
   }
   ```

4. Create `src/lib/db/index.ts` that re-exports:
   ```typescript
   export { db } from './client';
   export * from './schema';
   ```

5. Start Docker services: `docker compose up -d`

6. Push schema to database: `npx drizzle-kit push`
